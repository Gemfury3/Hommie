Technical Architecture & Design

1. Architecture Overview

The platform employs a Model-View-Template (MVT) architecture inherent to Django, providing clean separation of concerns and scalability:

Frontend (Bootstrap 4, HTML5, CSS3) ↓ URL Router (Django URLs) ↓ Views Layer (Business Logic) ↓ Models Layer (ORM Database Access) ↓ Database (SQLite3 → PostgreSQL in production)

2. Technology Stack

Layer Technology Version/Details
Framework Django 3.2.25 (LTS)
Language Python 3.12
Frontend Bootstrap 4.6
Database (Dev) SQLite3 Native
Database (Prod) PostgreSQL 14+
WSGI Server Gunicorn Latest
Static Files WhiteNoise Compressed manifest storage

3. Modular Application Structure

The project is organized into six discrete Django applications, each handling specific domain responsibilities:

  • Accounts: Custom user model, authentication, realtor profiles
  • Listings: Property listings, listing types, images, customer interactions
  • Core: Address management, geographic data, utility functions
  • Contacts: Contact forms, messaging, communication tracking
  • Documents: File management for listing attachments
  • Project: Django configuration, URL routing, middleware

4. Security Considerations

  • SECRET_KEY managed via environment variables (never hardcoded)
  • DEBUG mode controlled by environment (defaults to False in production)
  • CSRF protection enabled on all forms
  • Email-based authentication (not vulnerable to username enumeration)
  • Password hashing using PBKDF2 with Django's built-in system
  • HTTPS recommended for production deployment

Core Features & Functionality

1. User Authentication System

Implementation: Custom user model with email as primary authentication field. This design prevents username-based enumeration attacks and aligns with modern authentication best practices.
  • Email-based login (not username)
  • Role-based access control (staff, superuser, regular users)
  • Secure password reset via email tokens
  • Account activation/deactivation
  • User profile management

2. Property Listing Management

Comprehensive listing system with advanced property data structure:

  • Listing Types: Apartment, House, Maisonette, Loft, Commercial
  • Listing Details: Title, description, price, bedrooms, bathrooms, garage spaces, square footage, lot size, ceiling height
  • Availability: Listing for rent or sale designation, "available from" date
  • Status Tracking: Published/online flag, monument protection status
  • Media Support: Primary image plus unlimited supplementary images with lightbox gallery

3. Geolocation & Mapping

  • Address-based geocoding using Nominatim service
  • Google Maps integration for property location visualization
  • Graceful degradation when geolocation service unavailable
  • Timezone support (Europe/Berlin default)

4. Multi-Language Support

Platform operates in English and German with Django's built-in internationalization (i18n) framework. Language selection via URL prefix (/en/, /de/) or user preference.

5. Realtor Management

  • Professional realtor profiles with photo
  • Contact information and specialization
  • MVP (Most Valued Professional) designation
  • Hire date tracking for performance analysis
  • Direct integration with user accounts

6. Communication Features

  • Contact form for property inquiries
  • Realtor-customer messaging interface
  • Contact tracking and history
  • Email notification system (foundation for enhancement)

7. Admin Interface

Django's powerful admin panel customized for real estate operations:

  • Autocomplete fields for related data
  • Search functionality across all key fields
  • List display customization
  • Bulk actions support
  • Import/export capabilities

Database Structure & Models

1. Core Data Models

CustomUser Model

Email: EmailField (primary authentication) First Name, Last Name: CharField Phone: CharField Address: ForeignKey(Address) is_active, is_staff, is_superuser: Boolean date_joined, last_login: DateTime

Listing Model

Title: CharField(50) Description: TextField Price: DecimalField(20,2) Bedrooms, Bathrooms: PositiveIntegerField Garage: IntegerField Square Feet, Lot Size: FloatField Ceiling Height: FloatField Image: ImageField Listing For: Enum(Rent/Sell) Protected: Boolean (monument status) Is Published: Boolean Free From Date: DateField Related: ListingType, Realtor, Address, Images

Realtor Model

Name: CharField User: ForeignKey(CustomUser) Photo: ImageField Description: TextField Phone, Email: CharField Is MVP: Boolean Hire Date: DateField

Address Model

Street: CharField House Number: CharField ZIP Code: CharField City: CharField State: ForeignKey(State) Country: ForeignKey(Country) [via State] Coordinates: Auto-generated via Nominatim

2. Relationship Diagram

CustomUser ← Realtor → Listing ↓ Address → State → Country ↑ ListingImage (1:Many) ListingFile (1:Many) ↓ ListingToCustomer (M:M tracking)

3. Database Integrity Features

  • Foreign Key Constraints: PROTECT on critical relationships prevents orphaned records
  • Unique Constraints: Email uniqueness on CustomUser
  • Auto-increment IDs: Primary key auto-management
  • Timestamps: created_at, updated_at on all major models
  • Soft Deletes: is_published flag allows soft deletion of listings

3. Known Limitations & Workarounds

  • Social Authentication: django-allauth temporarily removed due to setuptools compatibility. Recommend upgrading for production OAuth integration.
  • Geolocation Service: External Nominatim dependency with timeout handling implemented. Production should cache results.
  • Missing Google Maps Key: Maps display degraded without API key. Easy fix with configuration.
  • Email Notifications: Foundation present but not fully implemented. Recommend adding Celery + Redis for production.
